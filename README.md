# commentingIsFun
Learn how to comment

Comments in a GitHub pull request are messages left by reviewers or teammates to discuss specific parts of the code. They are usually attached to a particular line or section of a change and are used to ask questions, suggest improvements, report potential bugs, or clarify the intent of the code. These comments help developers communicate during the review process and ensure the final code is clear, correct, and maintainable.

Comments in GitHub pull requests are important because they help improve code quality and collaboration within a team. By leaving comments, reviewers can point out bugs, suggest better approaches, and ensure the code follows project standards. PR comments also create a clear record of decisions and reasoning, which helps other developers understand the changes later. Overall, they make the development process more transparent, maintainable, and collaborative.

Resolving comments in a GitHub pull request helps keep the review process organized and ensures that feedback has been properly addressed. First, read the reviewer’s comment carefully and understand what change is being requested. Then update your code locally to fix the issue, commit the changes, and push them to the same branch of the pull request. After pushing, reply to the comment explaining what you changed so the reviewer can easily verify the fix. Once the issue is addressed and confirmed, you or the reviewer can mark the conversation as resolved, which keeps the PR discussion clean and shows that the feedback has been handled.
